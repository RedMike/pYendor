Class Hierarchy

  • lib.base.Application: Main application class.
  • lib.entity.Entity: Base entity class.
    • lib.entity.Item: Base non-blocking, visible, liftable and usable entity for subclassing.
      • lib.entity.Talker: Simple entity that prints a message on collision, collidable by default.
    • lib.entity.NPC
      • lib.entity.Beast: Subclassed beast entity, can be hit by player.
      • lib.entity.Player: Simple player class.
    • lib.entity.Trap: Base class for solid, collidable, unliftable, unusable ents for subclass.
      • lib.entity.Door: Simple door class openable only by player, subclass of Trap.
  • lib.entity.EntityMan: Simple entity manager class.
  • lib.map.Generator
    • lib.map.BasicGenerator
  • lib.interface.KeyboardListener
  • lib.map.Map
  • lib.object.Object
    • lib.object.Camera
  • lib.time.Scheduler
  • lib.entity.Tile: Tile utility class.
  • lib.map.Tile
  • object: The most base type
    • lib.graphics.RootWindow: Root window object that contains all other windows.
    • lib.graphics.Window: Basic window from which all are derived from.
      • lib.graphics.BorderedWindow: Window with a set tile border.
        • lib.graphics.BorderedMessageWindow: Main messaging window type.
          • lib.graphics.ChoiceWindow: Main menu type, single choice from multiple ones.
          • lib.graphics.GridWindow: Main inventory window type.
          • lib.graphics.InputWindow: One line input window.
          • lib.graphics.MultiChoiceWindow: Multi-choice window.
        • lib.graphics.DebugWindow: Simple debug window.
        • lib.graphics.LayeredGameWindow: Main game window, supports layers.
        • lib.graphics.StatusWindow: Variable bar-number bar window.
    • lib.graphics.WindowManager: Window manager class that handles Application <-> Window actions.
